# BILLIARD COACH - MVP 기획서

## 1. 개요

| 항목 | 내용 |
|---|---|
| 플랫폼 | Android (추후 iOS 확장) |
| 게임 종류 | 4구 (MVP) / 3구 (추후 추가, MVP 단계 잠금) |
| 당구대 종류 | 중대 (MVP) / 대대 (추후 추가, MVP 단계 잠금) |

## 2. 당구대 및 공 규격

### 2.1 중대 규격

| 항목 | 값 |
|---|---|
| 가로 (긴 쿠션 안쪽) | 2,540 mm (254 cm) |
| 세로 (짧은 쿠션 안쪽) | 1,270 mm (127 cm) |
| 가로:세로 비율 | 2:1 |
| 쿠션 높이 | 38 mm |

### 2.2 공 규격

| 항목 | 값 |
|---|---|
| 공 지름 | 65.5 mm |
| 공 개수 | 총 4개 |
| 구성 | 흰 공 1개, 노란 공 1개, 빨간 공 2개 |

### 2.3 쿠션 포인트 규격

| 쿠션 | 간격 | 포인트 개수 | 분할 |
|---|---|---|---|
| 장쿠션 (긴 쪽) | 317.5 mm | 7개 | 8등분 |
| 단쿠션 (짧은 쪽) | 317.5 mm | 3개 | 4등분 |

> 참고: 쿠션 높이(38mm)는 다이아몬드 포인트가 위치한 쿠션 면과 공이 놓이는 펠트(바닥) 면 사이의 높이 차이를 의미합니다. 카메라 각도에 따라 두 면 사이에 미세한 시차(parallax)가 발생할 수 있어, 탑뷰 변환 시 이 높이값을 보정 상수로 활용해 정밀도를 높입니다.

## 3. 화면 구성

### 3.1 [인트로]

- 타이틀: **BILLIARD COACH**
- 중대 / 대대 선택 (MVP: 대대 잠금)
- 4구 / 3구 선택 (MVP: 3구 잠금)
- 입장 버튼

### 3.2 [촬영] - 실시간 검출 + 피드백 (B안)

정지 이미지를 찍은 후 인식을 시도하는 방식이 아니라, **카메라 프리뷰 단계에서부터 실시간으로 당구대 상태를 분석하여 사용자에게 즉시 피드백을 주는 방식**을 채택합니다.

**동작 방식**

1. 카메라를 켜면 화면에 당구대 촬영을 위한 가이드 오버레이가 표시됩니다.
2. 카메라 프리뷰가 실행되는 동안, 온디바이스(On-device) 방식으로 매 프레임마다 쿠션의 다이아몬드 포인트를 실시간 검출합니다. (ML Kit / TFLite 등 온디바이스 모델 활용)
3. 검출된 포인트 개수가 최소 조건(예: 장쿠션 3점 이상 + 단쿠션 2점 이상)을 충족하면:
   - 가이드 오버레이 색상이 빨강/노랑 → **초록색**으로 전환
   - 촬영 버튼이 활성화되거나, 일정 시간 안정적으로 유지될 경우 **자동 촬영**
4. 최소 조건을 충족하지 못하면:
   - 가이드 오버레이는 비활성 색상 유지
   - 상황에 맞는 실시간 안내 문구 표시
     - 예: "조금 더 뒤로 가주세요"
     - 예: "당구대 전체가 잘 안 보여요"
     - 예: "카메라를 조금 더 위에서 비춰주세요"

**장점**
- 촬영 시점에 미리 인식 가능 여부를 걸러주기 때문에 촬영 성공률이 매우 높음
- 문서 스캔 앱(명함 스캔, 문서 스캔 등)과 유사한 UX 흐름이라 사용자에게 친숙하고 직관적임
- 촬영 후 "인식 실패, 다시 찍어주세요" 같은 사후 재촬영 빈도를 크게 줄일 수 있음

**단점 / 고려사항**
- 카메라 프리뷰 프레임마다 실시간으로 검출 로직을 돌려야 하므로 초기 개발 공수가 정적 이미지 방식보다 더 소요됨
- 온디바이스 실시간 처리이므로 기기 성능에 따라 프레임 처리 속도(FPS) 편차가 발생할 수 있어 최적화가 필요함
- 개발 순서상, 정지 이미지 기반의 포인트 검출 및 호모그래피 변환 로직을 먼저 안정화한 뒤, 동일 로직을 실시간 프리뷰에 적용하는 방식으로 단계적으로 확장하는 것이 효율적임

**전체가 촬영되지 않은 경우의 처리**
- 당구대 전체가 프레임에 다 들어오지 않고 일부만 촬영되더라도, 인식된 쿠션 다이아몬드 포인트들을 기준으로 전체 당구대 좌표를 역산하여 탑뷰(Top View)로 자동 변환합니다.

### 3.3 [메인]

**상단 (화면의 약 70~80%, 세로 방향)**
- 당구대 탑뷰 표시
- 상단에 놓여진 공 중 쳐야 하는 공 선택 가능

**하단 (탭 방식 선택)**
1. 일반
2. 3쿠션

- 각 탭 클릭 시: 촬영된 이미지의 공 위치에 해당하는 다양한 당구 길 명칭 버튼 표시 (예: 뒤돌려치기 등)
- 각 명칭 버튼 클릭 시:
  - 상단 당구대 탑뷰에 가이드 길 표시
    - 당점 이미지 표시
    - 타점(공 맞추는 위치) 이미지 표시
    - 키스 확률 % 표기
  - 상단 우측 **플레이 버튼**: 클릭 시 공 움직임 애니메이션 재생

**공통 기능**
- 메인 화면에 "다시 촬영하기" 버튼 배치

## 4. 화면 흐름 요약

```
인트로 (중대/4구 선택)
    ↓
촬영 (실시간 다이아몬드 포인트 검출 → 조건 충족 시 자동/수동 촬영 → 탑뷰 자동 변환)
    ↓
메인 (탑뷰 표시 → 공 선택 → 샷 종류 선택 → 가이드 표시 → 플레이)
    ↓
(다시 촬영하기 → 촬영 화면으로 복귀)
```

## 5. 개발 진행 순서 제안 (참고)

1. 정지 이미지 기반 다이아몬드 포인트 검출 + 호모그래피 변환 로직 안정화
2. 위 로직을 카메라 프리뷰(실시간 프레임)에 적용하여 B안(실시간 검출 + 피드백) 구현
3. 공 4개(흰/노랑/빨강×2) 위치 검출 로직 추가
4. 기본 샷 패턴(1~2개) 룰 기반 가이드 구현 및 검증
5. 샷 패턴 라이브러리 확장 + 키스 확률 근사치 로직 추가
6. 공 움직임 애니메이션(플레이 기능) 구현
7. 메인화면 UI/UX 고도화 (카드 캐러셀, 시점 자동 회전, 줌 인/아웃, 가로/세로 모드 지원)

## 6. 메인화면 UI/UX 고도화

### 6.1 샷 선택 - 카드 캐러셀 방식

기존 "버튼을 눌러 길이 표시되는 방식" 대신, 지도 앱의 다중 경로 안내와 유사한 **카드 스와이프 + 실시간 프리뷰 방식**을 채택합니다.

**동작 방식**
- 하단에 얇은 카드 캐러셀 배치, 각 카드에는 미니 경로 아이콘 + 샷 이름 표시 (예: 뒤돌려치기, 앞돌려치기 등)
- 카드를 스와이프하면 손을 떼지 않아도 상단 당구대 탑뷰에 해당 경로가 즉시 그려짐 ("훑어보면서 고른다"는 느낌)
- 특정 카드가 화면 중앙(또는 선택 영역)에 위치하면 자동으로 해당 샷이 "선택됨" 상태로 전환
- 카드 우측 상단 또는 캐러셀 옆에 플레이 버튼 배치, 선택된 샷의 공 움직임 애니메이션 재생

**장점**
- 버튼 클릭 방식보다 탐색적이고 세련된 인터랙션 제공
- 여러 샷 후보를 빠르게 비교하며 훑어볼 수 있음
- 지도 앱 등에서 이미 사용자에게 익숙한 UX 패턴이라 학습 곡선이 낮음

### 6.2 카드별 시점 자동 회전 (남/북 기준 정렬)

각 카드(샷 패턴)를 선택할 때, **해당 샷을 실제로 치는 사람의 위치를 기준으로 당구대 시점이 자동으로 회전**되도록 합니다.

**동작 방식**
- 각 샷 패턴마다 "치는 사람이 서는 위치(방향)"가 데이터로 정의되어 있음
- 카드 선택 시 당구대 탑뷰가 자동 회전되어, 항상 **치는 사람 기준 남쪽(화면 하단) / 공이 향하는 방향 기준 북쪽(화면 상단)**으로 정렬됨
- 즉, 사용자가 어느 방향에서 어떤 샷을 치든 항상 "내가 치는 방향이 아래에서 위로 향하는" 일관된 시점으로 표시됨

**목적 및 효과**
- 사용자가 실제 스탠스(서는 위치)에서 보는 것과 동일한 방향으로 가이드를 확인할 수 있어 직관성이 크게 향상됨
- 매 샷마다 사용자가 직접 화면을 돌려볼 필요 없이 자동으로 최적 시점이 제공됨

**구현 시 고려사항**
- 회전은 순간적으로 튀지 않도록 부드러운 애니메이션 전환(수백 ms 내외) 처리 필요
- 회전 중에도 가이드 길, 당점, 공 위치 등 모든 오버레이 요소가 테이블과 동일한 좌표계에서 함께 회전되어야 함 (테이블만 돌고 가이드가 고정되어 어긋나는 현상 방지)
- 수동 회전(사용자가 직접 두 손가락으로 회전) 기능과 카드 선택 시 자동 회전 기능이 충돌하지 않도록 우선순위 정의 필요 (예: 카드 선택 시 자동 회전 우선 적용, 이후 사용자가 수동으로 조정 가능)

### 6.3 줌 인 / 아웃

- 두 손가락 터치 후 벌리면 확대, 좁히면 축소되는 핀치 줌(pinch zoom) 제스처 지원
- 확대/축소된 상태에서도 가이드 길, 당점, 공 위치 등 모든 오버레이가 테이블과 동일한 비율로 함께 확대/축소되어야 함
- 과도한 확대/축소를 방지하기 위한 최소/최대 배율 제한 필요

### 6.4 가로 / 세로 모드 지원 (메인화면 한정)

- 인트로, 촬영 화면 등 다른 화면은 세로 모드로 고정
- **메인화면(당구대 탑뷰 확인 화면)에 한해** 가로 모드와 세로 모드를 모두 지원
- 세로 모드: 화면 상단 70~80%에 당구대 세로 배치, 하단에 카드 캐러셀
- 가로 모드: 당구대가 가로로 넓게 표시되며, 실제 당구대 비율(2:1)에 가깝게 보여 더 넓은 시야 확보 가능. 카드 캐러셀 및 컨트롤 UI는 가로 모드에 맞게 레이아웃 재배치 필요
- 회전 전환 시 현재 선택된 샷, 줌 배율, 시점 회전 상태 등이 유지되도록 상태 관리 필요
